Поле ввода используется для коротких текстовых значений или цифровых, не обозначающих сумму.
Для ввода более 5 слов используем многострочное поле ввода.
Вводимое значение может иметь определенный формат:
- Поле с паролем
- Поле с телефоном
- Поле с маской

В HTML5 поле ввода может быть разных типов — color, date, email, password, text, url и т.д. В сервисах Контура используйте только типы text и password. Остальные типы либо не поддерживаются основными браузерами, либо предлагают плохой пользовательский интерфейс и валидацию, которая будет отличаться по дизайну от нашей. (УТОЧНИТЬ У РАЗРАБОТЧИКОВ)

### Очистка поля
Крестик показывается, когда в поле введен хотя бы 1 символ. Крестик показывается даже когда фокус находится вне поля:
![[Pasted image 20240208140431.png|280]]

При клике на крестик фокус переходит в поле, введенное значение стирается:
![[Pasted image 20240208140535.png|280]]
*Область нажатия на крестик выделена фиолетовым цветом*

![[Pasted image 20240208140006.png|280]]

Internet Explorer 10 рисует свой крестик для любого поля ввода в фокусе. Чтобы этот крестик не перекрывал иконку, нужно дописать стиль:
`-ms-clear {`
	`display: none;`
`}`

### Счетчик количества введенных символов
Поле, в котором есть счетчик, ничем не отличается от обычного текстового поля.
Счетчик отображается только при состоянии `Focused`. По умолчанию отображается количество символов, которое можно ввести в поле.
![[Pasted image 20240207181612.png|280]]

Во время ввода символов счетчик изменяет значение и показывает, сколько символов можно ввести до его заполнения.
![[Pasted image 20240207182118.png|280]]

При вводе последнего символа из лимита счетчик показывает 0. Дальнейший ввод символов невозможен. При попытке ввода поле вспыхивает синим.

### Дизайн и отступы
Углы поля имеют скругления в 2px. Переменная: `border-radius: var(--roundness-input, 2px)`.
Ширина обводки в состоянии `Focused`, `Error` и `Warning` составляет 2px. Ширина обводки не должна влиять на смещение контента внутри, чтобы избежать дёргания поля.
